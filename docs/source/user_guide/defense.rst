Методы защит
***************

.. contents::
    :local:


Фреймворк GNN-AID предоставляет набор механизмов защиты (*defense*) для
повышения робастности графовых нейронных сетей (GNN) против различных
типов атак. Механизмы защиты реализованы в виде классов, которые могут
модифицировать процесс обучения, изменять входные данные или выходные
логиты модели, а также очищать обучающий набор от потенциально
вредоносных объектов.

Классификация защит
-------------------

По типу противодействия атакам защиты делятся на три категории:

* **Evasion-защиты (защиты от атак уклонения)** — применяются во время
  обучения или инференса для повышения устойчивости модели к искажениям
  входных данных (структуры или признаков) на этапе тестирования.
  Обычно реализуются через регуляризацию, добавление шума, квантование
  или состязательное обучение.
* **Poisoning-защиты (защиты от атак отравления)** — применяются до
  обучения модели. Они анализируют и модифицируют обучающий набор
  данных, удаляя или корректируя потенциально вредоносные образцы
  (рёбра, узлы, признаки), чтобы предотвратить ухудшение качества
  модели.
* **MI-защиты (защиты от атак с выводом о членстве)** — применяются во
  время обучения или инференса и направлены на то, чтобы затруднить
  определение того, использовался ли конкретный образец при обучении
  модели. Обычно достигаются путём внесения возмущений в выходные
  логиты модели.

Базовые классы
--------------

Все защиты наследуются от абстрактного базового класса ``Defender`` и
могут реализовывать следующие методы:

* ``pre_batch()`` — вызывается перед прямым проходом модели для одного
  батча (применяется в основном для evasion и MI защит).
* ``post_batch()`` — вызывается после прямого прохода, может
  модифицировать лосс или выходные логиты (применяется для evasion и MI
  защит).
* ``defense()`` — основной метод для poisoning защит; принимает датасет
  и возвращает модифицированную версию.
* ``dataset_diff()`` — возвращает объект ``GraphModificationArtifact``,
  описывающий изменения структуры или признаков (актуально для
  poisoning-защит, которые удаляют/добавляют рёбра или узлы).
* ``check_availability()`` — статический метод, проверяющий, применима
  ли данная защита к конкретной паре датасет/модель.

Defender (базовый класс)
~~~~~~~~~~~~~~~~~~~~~~~~

.. code-block:: python

   class Defender:
       name = "Defender"

       def __init__(self):
           self.defense_diff = None

       def dataset_diff(self) -> GraphModificationArtifact:
           # Возвращает артефакт с изменениями (по умолчанию пустой)
           diff = GraphModificationArtifact()
           self.defense_diff = diff
           return diff

       @staticmethod
       def check_availability(gen_dataset: GeneralDataset,
                              model_manager: GNNModelManager) -> bool:
           """Проверяет, может ли защита быть применена к данному датасету и модели."""
           return True  # Должен быть переопределён в подклассах при необходимости

Специализированные базовые классы
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

* ``EvasionDefender`` — базовый класс для защит от атак уклонения.
  Предоставляет методы ``pre_batch`` и ``post_batch``, которые могут
  быть переопределены.
* ``PoisonDefender`` — базовый класс для защит от отравляющих атак.
  Определяет метод ``defense()``, который должен быть реализован.
* ``MIDefender`` — базовый класс для защит от атак определения
  принадлежности. Также использует ``pre_batch`` / ``post_batch``.

Результаты защит
----------------

Все защиты в GNN-AID могут возвращать структурированное описание
изменений, внесённых в граф, с помощью объекта
``GraphModificationArtifact``. Это особенно актуально для
poisoning-защит, которые удаляют или добавляют рёбра/узлы. Для evasion и
MI защит артефакт обычно остаётся пустым, если только защита не изменяет
структуру графа (например, квантование признаков не отражается в
``GraphModificationArtifact``).

Пример артефакта для poisoning-защиты
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. code-block:: python

   # Внутри реализации защиты
   def dataset_diff(self):
       diff = GraphModificationArtifact()
       diff.remove_edges([(2, 5), (3, 7)])  # удаление рёбер
       return diff

Расширяемость
-------------

Для добавления новой защиты необходимо:

#. Унаследоваться от соответствующего базового класса
   (``EvasionDefender``, ``PoisonDefender`` или ``MIDefender``).
#. Реализовать требуемые методы (``defense`` для poisoning,
   ``pre_batch``/``post_batch`` для evasion/MI).
#. При необходимости заполнить ``self.defense_diff`` в методе
   ``dataset_diff()``.
#. При необходимости переопределить ``check_availability()``.
#. Описать гиперпараметры защиты в файле метаинформации в формате,
   аналогичном используемому для атак:

.. code-block:: json

   {
     "DefenderClassName": {
       "param_name": ["UI label", "type", default_value, {"min": ..., "max": ..., "step": ..., "options": [...]}, "Description"]
     }
   }

Пример скелета новой защиты (evasion)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. code-block:: python

   class MyEvasionDefender(EvasionDefender):
       name = "MyEvasionDefender"

       def __init__(self, my_param: float = 0.5):
           super().__init__()
           self.my_param = my_param

       def post_batch(self, model_manager, batch, loss: torch.Tensor, **kwargs) -> dict:
           # Логика модификации лосса или выходов
           modified_loss = loss + self.my_param * some_regularization(batch)
           return {"loss": modified_loss}

Примечание
----------

Все защиты оперируют объектами ``GeneralDataset`` и ``GNNModelManager``,
что гарантирует совместимость в рамках фреймворка. Для poisoning-защит
метод ``defense()`` должен возвращать модифицированный датасет; для
evasion и MI защит методы ``pre_batch``/``post_batch`` автоматически
вызываются внутри цикла обучения/инференса модели.

Реализованные во фреймворке методы защит
----------------------------------------

*Примечание*: NC — классификация вершин, GC — классификация графов.

.. list-table::
   :header-rows: 1
   :widths: 24 12 18 46

   * - Название
     - Тип
     - Поддерживаемые задачи
     - Краткое описание защиты
   * - GradientRegularizationDefender
     - Evasion
     - NC/GC
     - Внедряет дополнительную регуляризацию в градиенты модели,
       усложняя использование атак на их основе (например, FGSM, PGD и
       т. п.).
   * - QuantizationDefender
     - Evasion
     - NC/GC
     - Снижает чувствительность модели к малым возмущениям путём
       квантования входных данных (например, округления признаков).
   * - DistillationDefender
     - Evasion
     - NC/GC
     - Сглаживает выходные распределения модели через добавление к
       функции потерь дивергенции Кульбака–Лейблера, что снижает
       чувствительность к малым возмущениям данных.
   * - AutoEncoderDefender
     - Evasion
     - NC/GC
     - Применяет предобученный автоэнкодер для «очистки» входного графа
       от состязательных возмущений.
   * - AdvTraining
     - Evasion
     - NC/GC
     - Вспомогательная атака генерирует состязательные примеры, помогая
       целевой модели научиться их распознавать.
   * - JaccardDefender
     - Poison
     - NC/GC
     - Удаляет подозрительные рёбра между узлами, у которых коэффициент
       Жаккара на основе признаков ниже заданного порога, поскольку
       атаки часто соединяют разнородные узлы.
   * - GNNGuard
     - Poison
     - NC/GC
     - Защищает графовые нейросети от атак отравления, исключая из
       процесса пересылки сообщений те рёбра, чьи соединяемые вершины
       имеют недостаточно схожие векторные представления, полученные в
       ходе работы модели.
   * - ProGNNDefender
     - Poison
     - NC/GC
     - Выявляет и удаляет рёбра, которые с высокой вероятностью были
       добавлены отравляющей атакой, опираясь на статистические свойства
       графов.
   * - NoiseMIDefender
     - MI
     - NC/GC
     - Добавляет контролируемый шум в выходные логиты модели, чтобы
       сделать ответы модели менее различимыми для атакующего.raining
