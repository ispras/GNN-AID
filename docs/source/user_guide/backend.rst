1. Модуль предобработки данных

    - Назначение: улучшить качество данных, устранить потенциальные ошибки и привести их в формат, пригодный для эффективного обучения. А также, создать матрицу признаков и разметку для последующего обучения моделей машинного обучения
    - Функциональность: конвертация данных в необходимый формат, создания матрицы признаков, логических масок для разделения данных на обучающий, валидационный и тестовый наборы данных, создание разметки
    - Результат: готовый к использованию набор данных для обучения моделей машинного обучения.

2. Конструктор моделей

    - Назначение: предназначен для определения архитектуры и логики работы модели машинного обучения. А также, включает в себя получение дополнительной информации о модели. Например, получение весов модели, обнуление параметров и так далее
    - Функциональность: определение архитектуры модели, инициализация весов, описание логики вычисления выходов модели на основе входных данных, дополнительная информация о промежуточных состояниях модели и других ее неосновных характеристик
    - Результат: возвращает объект модели с заданной архитектурой, готовый к обучению и использованию для прогнозирования

3. Менеджер моделей

    - Назначение: изолирует процессы, связанные с управлением модели, от самой модели
    - Функциональность: обучение модели, сохранение и загрузка модели, валидация и тестирование модели, обработка гиперпараметров
    - Результат: обученная модель, готовая к использованию для решения поставленной задачи. А также, отчет о метриках, которые позволяют оценить производительность модели

4.  Модуль интерпретирования

    - Назначение: повысить прозрачность моделей, и дать пользователям возможность лучше понять, какие факторы влияют на итоговый прогноз модели
    - Функциональность: локальная и/или глобальная интерпретация модели, в зависимости от выбранного алгоритма, сохранение результатов интерпретации
    - Результат: интерпретация моделей, для лучшей прозрачность работы модели, что позволяет пользователям принимать более обоснованные решения, а также выявлять возможные ошибки или смещение в работе модели.
