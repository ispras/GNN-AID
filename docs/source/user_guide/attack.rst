Методы атак
**************

.. contents::
    :local:


Фреймворк GNN-AID предоставляет набор инструментов для проведения атак на
графовые нейронные сети (GNN), позволяющих оценивать их робастность к
различным возмущениям и попыткам извлечения информации. Все атаки
реализованы в виде классов, реализующих единый интерфейс. Возможно
создание и применение пользовательских атак, реализующих аналогичный
интерфейс.

Классификация атак
------------------

По цели и времени воздействия атаки делятся на три основных типа:

* **Evasion-атаки (атаки уклонения)** — применяются после обучения
  модели. Их цель — исказить входные данные (структуру графа или
  признаки) для конкретных объектов так, чтобы модель ошиблась при
  классификации.
* **Poisoning-атаки (атаки отравления данных)** — выполняются до
  обучения модели. Злоумышленник внедряет искажения в обучающие данные,
  чтобы ухудшить общую производительность модели.
* **Атаки определения принадлежности (Membership Inference, MI)** — не
  изменяют данные, а пытаются определить, использовался ли данный узел,
  ребро или граф при обучении модели.

По уровню доступа к модели атаки могут требовать знания внутренних
параметров (градиентов, скрытых представлений) — **белый ящик**, либо
работать только с входными данными и выходами модели — **чёрный ящик**.
Конкретные реализации атак могут относиться к любому из этих типов.

Базовые классы
--------------

Все атаки наследуются от абстрактного базового класса ``Attacker`` и
обязаны реализовывать следующие методы:

* ``attack()`` — основная логика атаки. Принимает объекты модели,
  датасета и аргументы, специфичные для задачи.
* ``dataset_diff()`` — возвращает объект ``GraphModificationArtifact``,
  описывающий изменения структуры или признаков (для evasion/poisoning
  атак).
* ``check_availability()`` — статический метод, проверяющий, применима
  ли атака к данной паре датасет/модель.

Атрибут ``name`` содержит человеко-читаемое имя атаки (используется в
логах и интерфейсе).

Attacker (базовый класс)
~~~~~~~~~~~~~~~~~~~~~~~~

.. code-block:: python

   class Attacker:
       name = "Attacker"
       def __init__(self): ...
       def attack(self, **kwargs): ...
       def dataset_diff(self) -> GraphModificationArtifact: ...
       @staticmethod
       def check_availability(gen_dataset, model_manager): ...

Специализированные базовые классы
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

* ``EvasionAttacker`` — базовый класс для атак уклонения, применяемых
  после обучения.
* ``PoisonAttacker`` — базовый класс для отравляющих атак, применяемых
  до обучения.
* ``MIAttacker`` — базовый класс для атак определения принадлежности к
  обучающей выборке.

Результаты атак
---------------

Все атаки в GNN-AID возвращают структурированные, сериализуемые
результаты, которые точно описывают, что и как было изменено. Формат
представления зависит от типа атаки.

Evasion и Poisoning атаки
~~~~~~~~~~~~~~~~~~~~~~~~~

Эти атаки возвращают объект ``GraphModificationArtifact`` —
легковесный контейнер, который фиксирует:

* добавленные / удалённые узлы (с возможными тензорами признаков);
* изменённые признаки узлов (поэлементные дельты);
* добавленные / удалённые рёбра (с возможными атрибутами рёбер).

Атаки определения принадлежности (MI)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Результаты сохраняются в ``MIResultsStore`` — объекте, подобном словарю,
который сопоставляет входные маски (например, ``train_mask``,
``test_mask``) с тензорами предсказанной принадлежности (булевы
значения). Каждая запись отвечает на вопрос: «Был ли данный образец
частью обучающей выборки?».

Расширяемость
-------------

Для добавления новой атаки необходимо:

#. Унаследоваться от соответствующего базового класса
   (``EvasionAttacker``, ``PoisonAttacker`` или ``MIAttacker``).
#. Реализовать метод ``attack()``.
#. Заполнить либо ``self.attack_diff`` (для атак, изменяющих
   структуру/признаки), либо ``self.results`` (для MI-атак).
#. При необходимости переопределить ``check_availability()``.
#. Описать все гиперпараметры атаки в файле метаинформации в следующем
   формате:

.. code-block:: json

   {
     "AttackClassName": {
       "param_name": ["UI label", "type", default_value, {"min": ..., "max": ..., "step": ..., "options": [...]}, "Description"]
     }
   }

Пример скелета новой атаки
~~~~~~~~~~~~~~~~~~~~~~~~~~

.. code-block:: python

   class MyCustomAttacker(EvasionAttacker):
       name = "MyCustomAttacker"

       def __init__(self, custom_param=1.0):
           super().__init__()
           self.custom_param = custom_param

       def attack(self, model_manager, gen_dataset, mask_tensor, **kwargs):
           # Логика атаки
           pass

       @staticmethod
       def check_availability(gen_dataset, model_manager):
           return gen_dataset.info.format == "ij"

Примечание
----------

Все атаки оперируют объектами ``GeneralDataset`` и ``GNNModelManager``,
что гарантирует совместимость в рамках фреймворка. Вам не нужно вызывать
метод ``.attack()`` напрямую — он автоматически запускается через
``GNNModelManager`` при необходимости.

Реализованные во фреймворке атаки
---------------------------------

*Примечание*: NC - классификация вершин, GC - классификация графов.

.. list-table::
   :header-rows: 1
   :widths: 18 10 16 16 40

   * - Название
     - Тип
     - Доступ к модели
     - Поддерживаемые задачи
     - Краткое описание атаки
   * - FGSM
     - Evasion
     - Белый ящик
     - NC/GC
     - Подбирает возмущение для входных данных методом скорейшего спуска
       в сторону антиградиента функции потерь исходной модели.
   * - Nettack
     - Evasion
     - Чёрный ящик
     - NC
     - Итеративно модифицирует рёбра целевого графа на основе прогнозов
       суррогатной модели.
   * - QAttack
     - Evasion
     - Чёрный ящик
     - NC
     - Основана на генетическом алгоритме и нацелена на ухудшение
       модулярности графа.
   * - PGD
     - Evasion
     - Белый ящик
     - NC/GC
     - Усовершенствованная версия FGSM, также использующая градиенты
       исходной модели для поиска оптимальных возмущений во входных
       данных.
   * - RLS2V
     - Evasion
     - Чёрный ящик
     - NC
     - Использует парадигму Q-обучения для атаки исходной модели,
       используя только информацию о предсказанных метках классов.
   * - ReWatt
     - Evasion
     - Чёрный ящик
     - NC
     - Переподключает рёбра в графе (одновременное удаление и
       добавление инцидентных рёбер для одной вершины), основывается на
       парадигме обучения с подкреплением.
   * - RandomPoisonAttack
     - Poison
     - Чёрный ящик
     - NC/GC
     - Простейшая отравляющая атака, которая случайным образом
       добавляет или удаляет рёбра в графе.
   * - MetaAttackFull
     - Poison
     - Чёрный ящик
     - NC/GC
     - Использует парадигму мета-обучения, подбирая итеративно
       отравленный датасет таким образом, чтобы минимизировать на нём
       качество работы суррогатной модели.
   * - MetaAttackApprox
     - Poison
     - Чёрный ящик
     - NC/GC
     - Облегчённая версия MetaAttackFull, использующая приближённые
       формулы для получения более выгодной временной асимптотики.
   * - CLGAAttack
     - Poison
     - Белый ящик
     - NC/GC
     - Изменяет входной граф на основе контрастивной функции потерь,
       использует градиенты используемой модели, однако подходит для
       задач обучения без учителя, так как не использует реальные метки
       классов.
   * - NaiveMIAttacker
     - MI
     - Чёрный ящик
     - NC/GC
     - Делает вывод о членстве объекта в обучающей выборке на основе
       высокой уверенности модели при прогнозировании его метки класса.
   * - ShadowModelMIAttacker
     - MI
     - Чёрный ящик
     - NC/GC
     - Обучает суррогатную модель, которая учится прогнозировать
       членство в обучающей выборке на основе ответов целевой модели.
