Архитектура веб-интерфейса
===========================

Веб-интерфейс реализован как одностраничное приложение (single page application) с помощью веб-фреймворка flask.
Взаимодействие с бэкендом выполняется с использованием Ajax — технологии асинхронных запросов к серверу, не влияющих на работу веб-страницы.
Отображение страницы в браузере и ее взаимодействие с пользователем управляется средствами HTML, CSS и JavScript.

Модель MVC
----------

Архитектурно, веб-интерфейс опирается на модель MVC (Model-View-Controller, Модель-Представление-Контроллер).
В данной схеме Модель представляет данные и методы работы с ними,
Представление отвечает за способ отображения данных пользователю на веб-странице,
Контроллер обрабатывает действия пользователя, давая модели команды на изменение данных.

Модель
~~~~~~

В случае GNN-AID данными являются:

* часть датасета, которая будет отображаться на экране
* архитектура и параметры GNN модели, загруженной или создаваемой пользователем
* параметры метода интерпретации (а затем и методов атаки и защиты)
* реестр датасетов, моделей и готовых результатов объяснений, доступных в хранилище

Модель реализует необходимые операции над данными, например, взятие окрестности заданной вершины, добавление сверточного слоя в архитектуру графовой нейросети.

Представление
~~~~~~~~~~~~~

Представление включает набор классов, которые описывают отображение конкретных элементов интерфейса: меню, граф датасета, информационная панель и т.д.
При необходимости, стили и внешний вид интерфейса можно поменять, переписав классы представления, при этом не затрагивая логику работы с данными.

В нашем случае представление также отвечает за взаимодействие с пользователем, если оно не влечет запросы к серверу.
Например, пользователь выбирает другой алгоритм расположения вершин отображаемого графа, и модель пересчитывает координаты вершин на экране в реальном времени.

Контроллер
~~~~~~~~~~

Контроллер отвечает за связь с бэкендом.
Например, пользователь выбирает новый датасет для загрузки, при этом контроллер создает соответствующий запрос к серверу,
а представление датасета приостанавливает отображение пока не будут получены и обработаны новые данные.

Технологии обмена данными
--------------------------

Как уже было сказано, для обмена данными используются Ajax запросы, а также веб-сокеты — двусторонние коммуникационные каналы клиент-сервер поверх TCP-соединения.

Ajax
~~~~

Ajax средства в GNN-AID применяются для разовых запросов типа загрузки датасета, модели или результатов объяснения.

Веб-сокеты
~~~~~~~~~~

Веб-сокеты более подходят для интерактивного взаимодействия и используются, например, для обновления информации во время обучения модели:
после каждой эпохи обучения GNN бэкенд посылает через веб-сокет новые значения весов модели,
вектора (эмбеддинги) вершин и значения метрик качества, которые сразу отрисовываются на экране.
